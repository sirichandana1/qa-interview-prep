TestNG Interview Questions and Answers

What is TestNG?

TestNG is a testing framework inspired by JUnit and NUnit but with additional features like test configuration, parallel execution, and data-driven testing.

How do you prioritize test cases in TestNG?

Use the priority attribute in @Test annotation. Lower values run first.

What is the difference between @BeforeMethod and @BeforeClass?

@BeforeMethod runs before each test method. @BeforeClass runs once before any methods in the class.

What is DataProvider in TestNG?

A feature used to pass multiple sets of data to a single test method.

How do you skip a test in TestNG?

Set enabled = false in the @Test annotation or throw SkipException.

Selenium WebDriver with Java Interview Questions and Answers

What is Selenium WebDriver?

Selenium WebDriver is an open-source tool used for automating web application testing.

What are the different types of locators in Selenium?

id, name, className, tagName, linkText, partialLinkText, cssSelector, xpath

Difference between findElement and findElements?

findElement: Returns a single WebElement.
findElements: Returns a list of WebElements.

How do you handle dropdowns in Selenium?
Select select = new Select(driver.findElement(By.id(\"dropdown\")));\nselect.selectByVisibleText(\"Option1\");
How do you handle alerts in Selenium?
Alert alert = driver.switchTo().alert();\nalert.accept();

API Testing Interview Questions and Answers

What is API Testing?

API Testing involves testing APIs to check their functionality, reliability, and security.

What tools are used for API Testing?

Postman, REST Assured, SoapUI, JMeter, Karate.

Difference between PUT and POST?

POST is for creating resources, PUT is for updating.

What are common status codes?

200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 404 Not Found, 500 Server Error

How do you test REST APIs?

Using GET, POST, PUT, DELETE requests and validating responses.

Manual Testing Interview Questions and Answers

What is Manual Testing?

Manual Testing is testing without automation tools.

Difference between Verification and Validation?

Verification ensures the product is built correctly, validation checks if the right product is built.

Levels of testing?

Unit, Integration, System, Acceptance

What is a test case?

A test case is a documented set of steps to verify functionality.

What is a bug life cycle?

New → Assigned → Open → Fixed → Retest → Verified → Closed
